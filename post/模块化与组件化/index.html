模块化与组件化
📅 2020-07-09 👦 Tomtom Young 📖 2926字 ⏱ 6分钟

1.传统模式

传统的开发模式就是一个页面对应1套html，js，css。这样就造成了3个问题：

  1. 命名冲突

    多个js文件如果存在同名的变量，则会发生变量覆盖，命名空间污染。

    1.js

    1const a=1

    2.js

    1const a=2

  2. 文件依赖

    各个js文件之间不能直接按需引用。必须都在.html文件中进行按顺序的引入。

  3. 代码冗余

    对于js来说，实现相似功能的代码是差不多甚至完全相同的。

    对于css和html来说，页面中的相同结构使用的代码是完全相同的。

2.js语言的局限性

js语言存在明显的缺陷性，因为js设计之初就没有想到会被用来实现各种复杂的功能。

Javascript的设计，只用了十天，这种语言的设计初衷，是为了解决一些简单的网页互动（比如，检查"用户名"是否填写），并没有考虑复杂应用的需要。设计者做梦也想不到，Javascript将来可以写出像Gmail这种极其庞大复杂的网页。

Javascript没有名称空间（namespace），很难模块化；没有如何将代码分布在多个文件的规范；允许同名函数的重复定义，后面的定义可以覆盖前面的定义，很不利于模块化加载。

3.模块化开发思想

ES6的发布让js语言具备了模块化的功能。

1.ES6模块化规范

ES6模块化规范是浏览器端和服务器端通用的模块化规范：

  1. 每一个js文件都是一个独立的模块
  2. 导入模块成员使用import关键字
  3. 暴露模块成员使用export关键字

2.ES6模块化语法

1.导入与导出

  1. 默认导出

    默认的导出语法是export default，导出默认的成员，以m1.js为例：

    m1.js

     1let a = 10
 2let b = 20
 3let c = 30
 4function show() {
 5 console.log('haha')
 6}
 7//每个模块中只允许使用一次export default
 8export default {
 9     a,
10    b,
11    show
12}

  2. 默认导入

    默认导入的语法是import 名称 from '路径'，以m2.js导入m1.js为例：

    m2.js

    1import m1 from './m1.js'
2console.log(m1)
3//输出结果为{ a:10 , b:20 , show:[Function:show]}

  3. 按需导出

    按需导出语法export let a=10

    m1.js

    1export  let a = 10
2export  let b = 'hahaha'
3export  function hello = function() { console.log('hello')}

  4. 按需导入

    按需导入语法import { a} from '路径'

    m2.js

    1import { a, b, hello} from './m1.js'
2console.log(a)
3console.log(b)
2.导入执行模块代码

有时候我们只想执行某个模块的代码，不需要得到里面向外暴露的成员，此时使用语法import '路径'，以m2.js导入m1.js为例：

m1.js

1for (let i=0;i<3;i++){
2    console.log(i)
3}

m2.js

1import './m1.js'

3.bable

bable在我理解应该是一种js语言的转换器，提高js在各个不同内核浏览器和不同版本浏览器的兼容性。

bable的官方定义：

Babel 是一个工具链，主要用于将 ECMAScript 2015+ 版本的代码转换为向后兼容的 JavaScript 语法，以便能够运行在当前和旧版本的浏览器或其他环境中。

1.安装bable
1npm install --save-dev @babel/core @babel/cli @babel/preset-env
2npm install --save @babel/polyfill
2.配置bable

新建babel.config.json文件，并写入以下内容：

 1{
 2  "presets": [
 3    [
 4      "@babel/env",
 5      {
 6        "targets": {
 7          "edge": "17",
 8          "firefox": "60",
 9          "chrome": "67",
10          "safari": "11.1",
11        },
12        "useBuiltIns": "usage",
13      }
14    ]
15  ]
16}

4.webpack

1.模块化开发困境

  1. 文件依赖关系错综复杂
  2. 静态资源请求效率低
  3. 对模块化的支持不够友好
  4. 浏览器对高级js兼容性差

2.webpack简介

Webpack 是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规则生成对应的静态资源。

5.webpack加载器

webpack默认只能打包js文件，非js文件webpack是处理不了的，需要调用loader加载器才能正常打包。loader加载器可以协助webpack打包特定的文件模块，比如：

  1. less-loader打包处理.less文件。
  2. sass-loader打包处理.sass文件。
  3. url-loader打包处理css中与url路径相关的文件。

6.vue-cli

vue-cli俗称脚手架，可以快速创建vue项目结构，使用vue-cli，我们可以快速生成vue项目，更便捷的使用组件化进行开发，而不需要我们一点点的去创建/管理/组织项目所需要的各种文件夹/文件。

1.创建vue项目

1vue creat 项目名

执行这条命令后，会进入一个交互式的面板，会让你选择项目需要的插件，以及各种配置，具体配置根据项目和个人使用习惯而有所区别。

2.项目目录说明

  1. node_modules

    项目依赖包。

  2. public

    存放静态资源。

    index.html：

    是一个模板文件，作用是生成项目的入口文件。浏览器访问项目的时候就会默认打开的是生成好的index.html。

  3. src 项目核心文件

    1. assets

      样式类文件，如css、less、sass以及外部的js文件都放在这里。

    2. components

      存放自定义组件的文件夹。

    3. App.vue

      根组件，所有页面都是在App.vue下进行切换的，也可以理解为所有的路由也是App.vue的子组件。

    4. main.js

      入口文件，主要作用是初始化vue实例，同时可以在此文件中引用某些组件库或者全局挂载一些变量。

  4. .babelrc

    babel配置参数。

  5. .gitignore

    git忽略文件。

  6. package-lock.json

    是在npm install时候生成的一份文件，用于记录当前状态下实际安装的各个npm package的具体来源和版本号。

  7. package.json

    模块基本信息项目开发所需要的模块，版本，项目名称。

  8. README.md 项目说明

  9. webpack.config.js webpack配置文件
